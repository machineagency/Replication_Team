## 📬 Submit a Project

Know an open source hardware project we should replicate? Submit a suggestion and we'll review it for our queue.

**[→ Open a Project Submission Issue](../../issues/new?template=project_submission.yml)**

We'll review submissions and triage them into the board. You'll be notified of status updates via the issue thread.

### What makes a good submission?
- The project is **open source** — hardware files, BOM, or documentation are publicly available
- There's a clear scientific or community use case
- It hasn't already been submitted (check [existing issues](../../issues) first)

---

## 🗂️ Completed Replications

Completed projects are documented in [`/replications`](./replications/) and linked from their closed issues. Each write-up includes:
- Build notes and deviations from the original design
- Audit findings and feedback sent to developers
- Bill of materials and sourcing notes
